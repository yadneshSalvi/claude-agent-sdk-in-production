# Part 1: Setup and Your First Agent

📖 Read along: [Claude Agent SDK in Production, Part 1](https://yadneshsalvi.com/blog/agent-sdk-1-first-agent)

🎬 See it run: **[demo.mp4](demo.mp4)** — a short screen recording of exactly what this part delivers.

A terminal agent that reads the Beanline CSVs and runs its own analysis to answer questions. No server, no UI yet — that's Parts 2 and 3. What's here:

- `backend/hello.py` — the smallest possible Agent SDK program: one `query()`, raw message objects printed as they arrive
- `backend/agent.py` — the Part 1 analyst: a workspace, a five-tool toolbox, `bypassPermissions`, and a readable trace of every tool call plus the cost of the run
- `backend/workspace/` — the Beanline sample data (`stores.csv`, `products.csv`, `sales.csv`), generated by [`data/generate_beanline.py`](../data/generate_beanline.py)

## Run it

You need [uv](https://docs.astral.sh/uv/) and Claude auth: either a Claude subscription you've logged into once (`claude` → `/login`), or an `ANTHROPIC_API_KEY` in your environment. The post covers both.

```bash
cd backend
uv sync
uv run python hello.py
uv run python agent.py
uv run python agent.py "Which product category makes the most money on weekends?"
```

`agent.py` runs with `permission_mode="bypassPermissions"` inside the `workspace/` folder — the agent can run any shell command your user can. That trade-off (and the Part 7–8 fix) is discussed honestly in the post.
